## Code of Conduct

This project adheres to a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior via GitHub Issues.

## How Can I Contribute?

### Reporting Bugs

Before creating a bug report, please check existing issues to avoid duplicates. When you create a bug report, include as many details as possible:

1. **Use a clear and descriptive title** for the issue
2. **Describe the exact steps to reproduce the problem**
3. **Provide specific examples** (code snippets, configuration files)
4. **Describe the behavior you observed** and what you expected
5. **Include your environment details**:
   - Python version
   - Operating system
   - Relevant package versions (LangGraph, CrewAI, MCP SDK)

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub Issues. When creating an enhancement suggestion:

1. **Use a clear and descriptive title**
2. **Provide a detailed description** of the proposed enhancement
3. **Explain why this enhancement would be useful** to learners
4. **Include examples** of how the enhancement would work

### Pull Requests

1. **Fork the repository** and create your branch from `main`
2. **Follow the coding style** used throughout the project
3. **Include comments** that explain WHY, not just WHAT
4. **Update documentation** if you change functionality
5. **Write clear commit messages** that describe your changes
6. **Test your changes** before submitting

#### Pull Request Process

1. Ensure your code follows the project's style guidelines
2. Update the README.md if your changes affect usage
3. Add yourself to the contributors list if this is your first contribution
4. The PR will be reviewed by maintainers and merged when approved

## Style Guidelines

### Python Code

- Follow PEP 8 style guidelines
- Use type hints where appropriate
- Include docstrings for functions and classes
- Keep functions focused and single-purpose

### Documentation

- Use clear, concise language
- Include code examples where helpful
- Follow Markdown best practices
- Ensure all links are valid

### Commit Messages

- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters
- Reference issues and pull requests when relevant

## Getting Help

If you need help or have questions:

1. **Check the course materials** first for answers
2. **Search existing issues** for similar questions
3. **Open a new issue** with the `question` label
